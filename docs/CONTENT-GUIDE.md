# Инструкция: загрузка контента (параграфы, тесты, изображения)

Как и в каком формате добавлять параграфы и тесты, где размещать изображения.

---

## 1. Структура папок

Контент хранится в папках по классу и предмету:

```
data/
├── <classId>/              Например: 2, 6
│   └── <subjectId>/        Например: math, history, english
│       ├── paragraphs.json   Параграфы (если нет content.json)
│       ├── quiz-*.json       Отдельные файлы тестов (если нет content.json)
│       ├── content.json      Либо один файл: параграфы + тесты (целевой формат)
│       └── media/            Папка для изображений предмета
│           ├── photo1.jpg
│           ├── vladimir.png
│           └── ...
```

- **classId** и **subjectId** должны совпадать с теми, что указаны в **`data.js`** (CONFIG), иначе предмет не появится в меню.

---

## 2. Два способа хранения данных

### Вариант A: Отдельные файлы (paragraphs.json + quiz-*.json)

Подходит, когда тесты ведёте отдельно от параграфов.

**Файл параграфов** — `data/<класс>/<предмет>/paragraphs.json`:

```json
[
  {
    "id": "19",
    "title": "§19. Название темы",
    "summary": "Краткое описание темы в 1–2 предложения.",
    "sections": [
      {
        "title": "Подзаголовок раздела",
        "content": "Текст раздела. Можно с переносами строк и разметкой: **жирный**, *курсив*."
      }
    ],
    "dates": [
      { "year": "862", "event": "Призвание варягов" }
    ],
    "terms": [
      { "term": "Термин", "definition": "Определение" }
    ],
    "people": [
      { "name": "Имя", "info": "Краткая информация" }
    ],
    "quizzes": [
      { "title": "Тест по §19", "file": "quiz-19.json" }
    ]
  }
]
```

**Файл теста** — `data/<класс>/<предмет>/quiz-19.json` (имя любое, на `.json`):

```json
{
  "title": "Тест по §19",
  "questions": [
    {
      "type": "choice",
      "q": "Текст вопроса?",
      "a": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
      "c": 0
    },
    {
      "type": "input",
      "q": "Введите год",
      "answer": "862"
    }
  ]
}
```

В параграфе в `quizzes` указываете **имя файла** теста (`"file": "quiz-19.json"`). Без этой ссылки тест не появится на странице темы.

### Вариант B: Единый content.json

Всё по предмету в одном файле — параграфы и тесты. Если в папке есть **`content.json`**, загрузчик использует только его (paragraphs.json и quiz-*.json не читаются).

**Файл** — `data/<класс>/<предмет>/content.json`:

```json
{
  "version": 1,
  "meta": {
    "classId": "6",
    "subjectId": "history",
    "title": "История",
    "updated": "2025-03-01"
  },
  "paragraphs": [
    {
      "id": "19",
      "title": "§19. Название",
      "summary": "Кратко.",
      "image": null,
      "sections": [
        { "title": "Раздел", "content": "Текст.", "image": null }
      ],
      "dates": [
        { "id": "d-1", "year": "862", "event": "Событие", "image": null }
      ],
      "terms": [
        { "id": "t-1", "term": "Термин", "definition": "Определение", "image": null }
      ],
      "people": [
        { "id": "h-1", "name": "Имя", "info": "Инфо", "image": null }
      ],
      "quizzes": [
        { "id": "quiz-19", "title": "Тест по §19", "source": "manual" }
      ]
    }
  ],
  "quizzes": [
    {
      "id": "quiz-19",
      "title": "Тест по §19",
      "source": "manual",
      "difficulty": "medium",
      "questions": [
        { "type": "choice", "q": "Вопрос?", "a": ["A", "B", "C"], "c": 0 }
      ]
    }
  ]
}
```

В параграфе в `quizzes` указываете **id** теста из корневого массива `quizzes` (например `"id": "quiz-19"`).

## 3. Изображения в параграфах

### Где размещать файлы

Все картинки предмета кладите в папку:

**`data/<класс>/<предмет>/media/`**

Примеры:
- `data/6/history/media/vladimir.jpg`
- `data/6/history/media/map-882.png`
- `data/2/math/media/schema.png`

Рекомендуемые форматы: **JPEG, PNG, WebP**. Имена файлов — латиница, цифры, дефис/подчёркивание (без пробелов и спецсимволов).

### Как указать изображение в JSON

В поле **`image`** пишите **путь относительно папки предмета** — то есть начинающийся с **`media/`** и имени файла.

| Где | Поле | Пример значения |
|-----|------|------------------|
| Параграф (обложка/картинка темы) | `paragraph.image` | `"media/topic.jpg"` |
| Раздел параграфа | `sections[].image` | `"media/section1.png"` |
| Дата/событие | `dates[].image` | `"media/event-862.jpg"` |
| Термин | `terms[].image` | `"media/term.png"` |
| Персона | `people[].image` | `"media/vladimir.jpg"` |

Если изображения нет — укажите **`null`** или не указывайте поле.

**Пример параграфа с картинками:**

```json
{
  "id": "21",
  "title": "§21. Крещение Руси",
  "summary": "...",
  "image": "media/kiev.jpg",
  "sections": [
    {
      "title": "Владимир",
      "content": "Текст...",
      "image": "media/vladimir.jpg"
    }
  ],
  "dates": [
    { "year": "988", "event": "Крещение Руси", "image": "media/988.jpg" }
  ],
  "terms": [
    { "term": "Крещение", "definition": "...", "image": null }
  ]
}
```

Итог: файл лежит в `data/6/history/media/vladimir.jpg`, в JSON — `"image": "media/vladimir.jpg"`.

## 4. Формат тестов (вопросы)

Каждый вопрос в `questions` — объект с полями:

**Вопрос с выбором ответа (choice):**

```json
{
  "type": "choice",
  "q": "Текст вопроса?",
  "a": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
  "c": 0,
  "hint": "Подсказка при ошибке (необязательно)",
  "explanation": "Объяснение (необязательно)"
}
```

- **q** — текст вопроса.
- **a** — массив вариантов (строки).
- **c** — индекс правильного варианта (0, 1, 2, …).
- **hint**, **explanation** — по желанию.

**Вопрос с вводом ответа (input):**

```json
{
  "type": "input",
  "q": "Введите год",
  "answer": "988",
  "hint": "Подсказка (необязательно)"
}
```

Или несколько допустимых ответов:

```json
{
  "type": "input",
  "q": "Назовите князя",
  "answers": ["Владимир", "Владимир Святославич"]
}
```

Проверка: без учёта регистра и с обрезкой пробелов.

---

## 5. Разметка текста в параграфах

В полях **текста** (summary, sections[].content и т.п.) можно использовать:

- **Перенос строки** — в JSON строка с символом новой строки `\n` или многострочная строка.
- **Жирный** — `**текст**`.
- **Курсив** — `*текст*`.

Пример:

```json
"content": "Первый абзац.\n\nВторой абзац. Важно: **дата 988** и *термин*."
```

## 6. Добавление нового класса или предмета

1. Создайте папку: **`data/<classId>/<subjectId>/`** (например `data/7/biology/`).
2. Положите туда **paragraphs.json** или **content.json** (и при необходимости **quiz-*.json**).
3. Откройте **`data.js`** и добавьте класс/предмет в **CONFIG**:

```javascript
const CONFIG = {
  classes: {
    "7": {
      name: "7 класс",
      subjects: {
        biology: {
          name: "Биология",
          path: "data/7/biology/"
        }
      }
    }
  }
};
```

Путь **path** должен заканчиваться слэшем и совпадать с папкой: `data/7/biology/`.

---

## 7. Краткая памятка

| Что | Куда | Формат |
|-----|------|--------|
| Параграфы | `data/<класс>/<предмет>/paragraphs.json` или внутри `content.json` | Массив объектов с id, title, summary, sections, dates, terms, people, quizzes |
| Тесты | `data/<класс>/<предмет>/quiz-*.json` или внутри `content.json` в `quizzes` | Объект с title, questions[] |
| Связь теста с параграфом | В параграфе: `quizzes: [{ "title": "...", "file": "quiz-19.json" }]` или в content: `quizzes: [{ "id": "quiz-19", "title": "..." }]` | file или id |
| Фото параграфа/раздела/даты/термина/персоны | Папка `data/<класс>/<предмет>/media/` | Файлы .jpg, .png, .webp |
| Ссылка на фото в JSON | В нужном объекте поле `"image": "media/имя-файла.jpg"` | Путь от папки предмета |

После изменения JSON обновите страницу или в консоли выполните **`Api.clearCache()`**, затем обновите экран.
