# Безопасность

Краткий обзор мер безопасности в School Portal и рекомендации.

---

## Репозиторий и окружение

- `admin.html` раздается как статическая страница, но **доступ к редактированию** контролируется проверкой роли `admins/{uid}` в Firestore.
- В открытом доступе — только портал для учеников (статика + JSON). Без admin-доступа панель не даст читать/записывать контент.

---

## XSS (внедрение скриптов)

- **Вывод контента:** все строки из JSON и из localStorage перед вставкой в разметку проходят через **`escapeHtml()`** (`assets/js/utils/safeHtml.js`). Используется `textContent` и замена кавычек, что нейтрализует теги и атрибуты.
- **Обработчики:** клики и действия реализованы через **`addEventListener`** и атрибуты **`data-*`** (например, `data-class-id`, `data-quiz-slug`). В разметку не подставляется код из данных (нет `onclick="..."` с подстановкой строк).
- **Форматирование параграфов:** `formatParagraphText()` сначала экранирует HTML, затем применяет ограниченную разметку (`**жирный**`, `*курсив*`, переносы строк). Произвольный HTML из контента не выполняется.

---

## Загрузка данных (API)

- **Проверка путей:** в `api.js` перед загрузкой проверяется допустимость пути:
  - запрещены пути, содержащие **`..`** (выход за пределы ожидаемой структуры);
  - запрещены пути, начинающиеся с **`http:`**, **`https:`** или **`//`** (исключается загрузка произвольных внешних URL).
- Запросы выполняются только к относительным путям в рамках текущего origin (статический хостинг). Это снижает риски обхода и SSRF при подмене контента (например, через localStorage в dataProvider).

---

## URL изображений

- В тестах и параграфах URL картинок формируются через **`safeImageSrc()`** (quiz.js) и **`imageHtml()`** (paragraph.js):
  - блокируются протокол **`javascript:`**;
  - для **`data:`** разрешены только URL вида **`data:image/...`** (остальные, в т.ч. `data:text/html`, отбрасываются);
  - пути с **`..`** не допускаются;
  - допускаются относительные пути и абсолютные `https?://` (внешние картинки — на совести источника контента).
- Значения подставляются в `src` только после проверки; атрибут дополнительно экранируется через `escapeHtml` где применимо.

---

## Рекомендации при развёртывании

1. **Content-Security-Policy (CSP):** при публикации на сервере рекомендуется настроить заголовок CSP (запрет inline-скриптов, ограничение источников загрузки). Это дополнительно страхует от внедрения скриптов при возможных ошибках в выводе.
2. **HTTPS:** при доступе по сети использовать HTTPS, чтобы данные и скрипты не подменялись по пути.
3. **Контент:** JSON и контент из localStorage считаются частично доверенными (могут правиться через админку). Санитизация и экранирование на стороне портала обязательны; при добавлении новых полей или типов контента их нужно выводить только через безопасные функции.
4. **Админка:** не раздавать `admin.html` и скрипты админки в публичную среду; использовать только локально или в защищённой зоне.

---

## Доступность (a11y)

- Для важных сообщений и подсказок используется контейнер с **`aria-live="polite"`** где применимо.
- Кнопки и интерактивные элементы имеют осмысленные подписи и при необходимости **`aria-label`** (например, «Выйти из теста»).
- В CSS учтён **`:focus-visible`** для навигации с клавиатуры.
