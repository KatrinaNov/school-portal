# School Portal — учебная тестовая система

Веб-приложение для просмотра учебных параграфов и прохождения тестов по выбранным темам. Поддерживает несколько классов и предметов, кастомные тесты из выбранных параграфов.

---

## Возможности

- **Параграфы** — загрузка и отображение тем с разделами, датами, терминами и персоналиями.
- **Тесты** — прохождение тестов по типам вопросов: выбор варианта (choice) и ввод ответа (input).
- **Свой тест** — выбор параграфов и формирование теста из объединённых вопросов (до 20 случайных).
- **Рандомизация** — Fisher–Yates shuffle для честной случайной выборки вопросов.
- **Админ-панель** — для локального использования (в этом репозитории не публикуется): управление классами, предметами, параграфами и тестами, экспорт/импорт JSON в localStorage.

## Формат JSON

### Параграфы (`paragraphs.json`)

Массив объектов параграфа:

```json
[
  {
    "id": "17",
    "title": "§17. Название темы",
    "summary": "Краткое описание.",
    "sections": [
      { "title": "Подзаголовок", "content": "Текст раздела." }
    ],
    "dates": [
      { "year": "1253", "event": "Событие" }
    ],
    "terms": [
      { "term": "Термин", "definition": "Определение" }
    ],
    "people": [
      { "name": "Имя", "info": "Краткая информация" }
    ],
    "quizzes": [
      { "title": "Тест по §17", "file": "quiz-17.json" }
    ]
  }
]
```

Поля `sections`, `dates`, `terms`, `people`, `quizzes` — опциональны.

### Тест (`quiz-*.json`)

```json
{
  "title": "Название теста",
  "questions": [
    {
      "type": "choice",
      "q": "Текст вопроса?",
      "a": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
      "c": 1,
      "hint": "Подсказка при ошибке (опционально)",
      "explanation": "Объяснение (опционально)"
    },
    {
      "type": "input",
      "q": "Введите год",
      "answer": "1253",
      "hint": "Подсказка при ошибке (опционально)"
    },
    {
      "type": "input",
      "q": "Введите ответ",
      "answers": ["вариант 1", "Вариант 2", "вариант 3"],
      "explanation": "Объяснение (опционально)"
    }
  ]
}
```

- **choice**: `c` — индекс правильного варианта (0-based). `a` — массив вариантов.
- **input**: `answer` (строка) или `answers` (массив допустимых ответов). Проверка без учёта регистра и с обрезкой пробелов.
- **hint / explanation** — показываются при неверном ответе; после 3 ошибок для input показывается правильный ответ и кнопка «Пропустить вопрос».


## Тестирование

### Unit-тесты (Jest)

Установка зависимостей:

```bash
npm install
```

Запуск тестов:

```bash
npm test
```

Покрытие (если настроено в `jest.config.js`):

```bash
npm run test:coverage
```

Тестируются:

- **normalizeAnswer** — обрезка пробелов, приведение к нижнему регистру, обработка `null`/`undefined`.
- **validateAnswer** — choice по индексу; input по строке и по массиву допустимых ответов; разный регистр и пробелы.
- **Логика 3 попыток** — `canShowSkip(attempts, maxAttempts)` и сброс попыток при смене вопроса (через тесты движка).
- **Логика skip** — что при пропуске не увеличивается счётчик правильных (тест сценария движка/состояния).
- **shuffleArray** — длина и состав элементов сохраняются; выборка 20 вопросов из большего списка (длина 20, все элементы из исходного массива).
- **getCorrectAnswerForDisplay** — для input возвращает первый допустимый ответ; защита от отсутствующих полей.

Движок (`quizEngine.js`) экспортирует функции для Node (Jest) и в браузере доступен через `window.QuizEngine` (или один общий namespace), чтобы не ломать текущую загрузку скриптов без сборки.
