# School Portal

Учебный тестовый портал: параграфы по темам и тесты с разными типами вопросов.

---

## Возможности

- **Классы и предметы** — навигация по классам и предметам из `data.js` (CONFIG).
- **Параграфы** — темы с разделами, датами, терминами, персонами и изображениями.
- **Тесты** — типы вопросов:
  - **choice** — один правильный вариант (текст и/или картинка);
  - **input** — ввод ответа (один или несколько допустимых);
  - **multiple_choice** — несколько правильных вариантов;
  - **match** — сопоставление пар (выбор из списка);
  - **fill_words** — вставка пропущенных слов (пошаговая проверка).
- **Изображения** — в вопросе и в вариантах ответа (пути относительно папки предмета).
- **Свой тест** — выбор параграфов и тип: по тестам параграфов, по датам, по понятиям, по персонам или комбинированный; лимит вопросов, рандомизация (Fisher–Yates).
- **Режим проверки** — после завершения теста показ правильных/неправильных ответов по типам (в т.ч. match и fill_words).
- **Адаптивная вёрстка** — мобильные экраны, safe-area, фокус для клавиатуры.
- **Админ-панель** (отдельная страница, локально) — управление контентом, экспорт/импорт в localStorage.

## Запуск

Статические файлы: откройте `index.html` в браузере или поднимите любой HTTP-сервер из корня проекта.

```bash
# Пример (npx без установки):
npx serve .
# или
npx http-server -p 8080
```

Переход по разделам — через hash: `#/`, `#/class/6`, `#/class/6/history`, `#/class/6/history/paragraph/17`, `#/class/6/history/quiz/quiz-19`.

---

## Webpack (рекомендуемый режим)

Разработка:

```bash
npm install
npm run dev
```

Сборка (статический деплой):

```bash
npm run build
```

Результат — папка `dist/`.

## Firebase (Auth + Firestore)

- Конфиг Firebase: `src/firebase.js`
- Авторизация/регистрация (Google + email): `src/auth.js`
- Профиль студента со статистикой: `#/me`
- Админка: `admin.html` (требует роль admin)
- Миграция данных из JSON в Firestore: `npm run migrate:firestore`

Подробно: `docs/FIREBASE.md`.


## Формат данных

- **Один файл по предмету:** `data/<класс>/<предмет>/content.json` с полями `version`, `meta`, `paragraphs`, `quizzes` (см. `docs/data-schema.md`, `docs/CONTENT-GUIDE.md`).
- **Либо отдельно:** `paragraphs.json` + файлы `quiz-*.json` в той же папке.
- Конфиг классов и предметов — в `assets/js/data.js` (CONFIG). Новый предмет нужно добавить в CONFIG и создать папку в `data/`.

## Тестирование

```bash
npm install
npm test
npm run test:coverage
```

Тесты: `assets/js/__tests__/` (Jest). Проверяются движок тестов (нормализация, валидация, shuffle) и генераторы вопросов.

## Безопасность

- Вывод в HTML — через `escapeHtml()` (защита от XSS).
- Обработчики — через `addEventListener` и `data-*`, без подстановки кода в разметку.
- Пути загрузки (getParagraphs, getQuiz) проверяются: запрещены `..` и абсолютные URL (защита от обхода и SSRF).
- URL изображений: запрещены `javascript:`, неразрешённые `data:` (разрешены только `data:image/...`), пути с `..`.

Подробнее: `docs/SECURITY.md`.
