# School Portal — учебная тестовая система

Веб-приложение для просмотра учебных параграфов и прохождения тестов по выбранным темам. Поддерживает несколько классов и предметов, кастомные тесты из выбранных параграфов.

---

## Возможности

- **Параграфы** — загрузка и отображение тем с разделами, датами, терминами и персоналиями.
- **Тесты** — прохождение тестов по типам вопросов: выбор варианта (choice) и ввод ответа (input).
- **Свой тест** — выбор параграфов и формирование теста из объединённых вопросов (до 20 случайных).
- **Рандомизация** — Fisher–Yates shuffle для честной случайной выборки вопросов.
- **Админ-панель** (`admin.html`) — управление классами, предметами, параграфами и тестами без сервера: данные в localStorage, экспорт/импорт JSON.

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│  UI Layer                                                        │
│  (index.html, #app, #breadcrumbs, main.css)                     │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Render Layer                                                    │
│  app.js: renderHome, renderClass, renderSubject, setBreadcrumbs  │
│  paragraph.js: openParagraph (рендер параграфа + тесты)            │
│  quiz.js: рендер вопроса, результата, кнопок (через getContainer)│
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Quiz Engine (quizEngine.js)                                      │
│  Чистая логика: normalizeAnswer, validateAnswer, shuffleArray,   │
│  getCorrectAnswerForDisplay, canShowSkip, MAX_ATTEMPTS             │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  State                                                           │
│  state.js: currentClass, currentSubject, path (для крошек)        │
│  quiz: текущий вопрос, correct/wrong/skipped, attempts (в замыкании)│
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Data Layer (api.js)                                             │
│  getParagraphs(path), getQuiz(path) + in-memory cache            │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Config (data.js)                                                │
│  CONFIG.classes[classId].subjects[subjectId].path                │
└─────────────────────────────────────────────────────────────────┘
```

- **Data Layer** — единственная точка загрузки JSON; кэш по ключу (путь к файлу).
- **State** — глобальный контекст навигации (класс/предмет/путь); состояние квиза инкапсулировано внутри сессии теста.
- **Render** — функции заполняют `#app` и `#breadcrumbs`; минимизация прямого доступа к DOM в логике квиза (один контейнер, делегирование).
- **Quiz Engine** — чистые функции без DOM; тестируются unit-тестами; в браузере подключается как `QuizEngine` (или через `window`).

## Формат JSON

### Параграфы (`paragraphs.json`)

Массив объектов параграфа:

```json
[
  {
    "id": "17",
    "title": "§17. Название темы",
    "summary": "Краткое описание.",
    "sections": [
      { "title": "Подзаголовок", "content": "Текст раздела." }
    ],
    "dates": [
      { "year": "1253", "event": "Событие" }
    ],
    "terms": [
      { "term": "Термин", "definition": "Определение" }
    ],
    "people": [
      { "name": "Имя", "info": "Краткая информация" }
    ],
    "quizzes": [
      { "title": "Тест по §17", "file": "quiz-17.json" }
    ]
  }
]
```

Поля `sections`, `dates`, `terms`, `people`, `quizzes` — опциональны.

### Тест (`quiz-*.json`)

```json
{
  "title": "Название теста",
  "questions": [
    {
      "type": "choice",
      "q": "Текст вопроса?",
      "a": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
      "c": 1,
      "hint": "Подсказка при ошибке (опционально)",
      "explanation": "Объяснение (опционально)"
    },
    {
      "type": "input",
      "q": "Введите год",
      "answer": "1253",
      "hint": "Подсказка при ошибке (опционально)"
    },
    {
      "type": "input",
      "q": "Введите ответ",
      "answers": ["вариант 1", "Вариант 2", "вариант 3"],
      "explanation": "Объяснение (опционально)"
    }
  ]
}
```

- **choice**: `c` — индекс правильного варианта (0-based). `a` — массив вариантов.
- **input**: `answer` (строка) или `answers` (массив допустимых ответов). Проверка без учёта регистра и с обрезкой пробелов.
- **hint / explanation** — показываются при неверном ответе; после 3 ошибок для input показывается правильный ответ и кнопка «Пропустить вопрос».

---

## Механика «3 попытки → пропустить»

- Применяется **только к вопросам типа input**.
- **maxAttempts = 3** — константа в движке.
- При каждом неверном ответе счётчик попыток для текущего вопроса увеличивается.
- После **3-й ошибки**:
  - показывается правильный ответ (первый из `answers` или значение `answer`);
  - появляется кнопка **«Пропустить вопрос»** (одна, без дублирования);
  - ввод блокируется (поле и кнопка «Ответить» отключаются).
- При нажатии **«Пропустить вопрос»**:
  - переход к следующему вопросу;
  - в счётчик правильных ответов балл **не** добавляется;
  - счётчик попыток сбрасывается для следующего вопроса.
- При переходе к следующему вопросу (правильный ответ или пропуск) попытки для нового вопроса всегда начинаются с 0.
- Утечки состояния не допускаются: попытки хранятся в замыкании сессии теста, сброс — только при смене вопроса.

## Кэширование

- **Где**: в `api.js`, объект `cache` внутри замыкания.
- **Ключ**: полный путь к файлу (`path + "paragraphs.json"` или путь к `quiz-*.json`).
- **Поведение**: первый запрос по ключу выполняет `fetch`, результат сохраняется; последующие вызовы с тем же ключом возвращают сохранённые данные без запроса.
- **Область**: одна вкладка; при возврате «Назад» к списку тем или к параграфу повторный запрос не выполняется.

## Тестирование

### Unit-тесты (Jest)

Установка зависимостей:

```bash
npm install
```

Запуск тестов:

```bash
npm test
```

Покрытие (если настроено в `jest.config.js`):

```bash
npm run test:coverage
```

Тестируются:

- **normalizeAnswer** — обрезка пробелов, приведение к нижнему регистру, обработка `null`/`undefined`.
- **validateAnswer** — choice по индексу; input по строке и по массиву допустимых ответов; разный регистр и пробелы.
- **Логика 3 попыток** — `canShowSkip(attempts, maxAttempts)` и сброс попыток при смене вопроса (через тесты движка).
- **Логика skip** — что при пропуске не увеличивается счётчик правильных (тест сценария движка/состояния).
- **shuffleArray** — длина и состав элементов сохраняются; выборка 20 вопросов из большего списка (длина 20, все элементы из исходного массива).
- **getCorrectAnswerForDisplay** — для input возвращает первый допустимый ответ; защита от отсутствующих полей.

Движок (`quizEngine.js`) экспортирует функции для Node (Jest) и в браузере доступен через `window.QuizEngine` (или один общий namespace), чтобы не ломать текущую загрузку скриптов без сборки.

## Админ-панель

Откройте `admin.html` в браузере. Всё работает без сервера и БД.

- **Классы** — добавить, переименовать, удалить.
- **Предметы** — добавить, переименовать, удалить, привязка к классу (путь к данным).
- **Параграфы** — редактирование id, title, summary, sections, dates, terms, people, привязка тестов; превью; проверка уникальности id.
- **Тесты** — добавление теста и вопросов (тип choice с вариантами и правильным ответом, тип input с допустимыми ответами); перетаскивание порядка вопросов.
- **Сохранение** — кнопка «Сохранить» пишет данные в `localStorage`. Главная страница (`index.html`) при загрузке подхватывает эти данные через `dataProvider.js` и показывает контент из админки вместо статических JSON.
- **Экспорт / Импорт** — скачать всё в JSON-файл или загрузить файл с валидацией структуры. Перед удалением и при несохранённых изменениях запрашивается подтверждение.

Формат единого JSON: `{ "config": { "classes": { ... } }, "paragraphs": { "data/2/math/": [ ... ] }, "quizzes": { "data/2/math/quiz-1.json": { "title": "...", "questions": [ ... ] } } }`.

## Зависимости

- Для работы приложения в браузере: только статические файлы (HTML, CSS, JS), без сборки.
- Для unit-тестов: Node.js и Jest (см. `package.json` и `jest.config.js`).

---

## Безопасность и доступность

- Вывод пользовательского контента из JSON экранируется через `escapeHtml` (защита от XSS).
- Кнопки и навигация по тесту используют `addEventListener` и `data-*` атрибуты, без подстановки кода в `onclick`.
- Сообщения об ошибках и подсказки выведены в контейнер с `aria-live="polite"` для скринридеров.
